# El Arte de Sentir el Entorno: Sensores y Evolución

En la ingeniería agéntica tradicional, solemos tratar a la IA como un procesador de texto. En **Harness Engineering v3.0**, la tratamos como un **Ser Sintiente en un Entorno de Datos**.

## 1. Más allá del Mapa: El Oído de la IA
Un agente no puede ser soberano si solo ve coordenadas. En la Abadía, el sonido de una campana o los pasos de un monje invisible son **Datos de Misión Crítica**.
- **Lección:** Diseña tus arneses para que incluyan "sensores" que traduzcan señales ruidosas (como un array de IDs de sonido) en contexto semántico.

## 2. Aprendizaje Evolutivo por Ejecución (Test-Time Learning)
No intentamos programar a Guillermo para que sepa qué significa cada sonido desde el primer día. Usamos el **Bucle de Experiencia**:
1. **Ejecución:** Guillermo oye un "beep" y no hace nada.
2. **Fallo:** El Abad lo pilla y lo expulsa.
3. **Dreamer:** El sistema analiza el log: "Hubo un Sonido ID 4 antes de la expulsión".
4. **Ratchet:** El Vault se actualiza: "Sonido ID 4 = Campana de Vísperas. Prioridad: Volver a la Iglesia".

## 3. Sentir el Entorno como Arnés de Seguridad
Sentir el entorno no es solo para "jugar mejor", es un mecanismo de **Seguridad Agéntica**. Si el agente "siente" que su novicio Adso está asustado o que la lámpara se apaga, su jerarquía de valores cambia automáticamente. La seguridad no se programa como una regla fija, sino como una **reacción sensorial**.
